# 音频合成自动化链路

## 目录结构

```
D:\AIBLI\audio_synthesis\
├── character_profile\          # 角色人设资产
│   ├── 解说员\                 # 角色文件夹（名字自定义）
│   │   ├── gpt_weights\        # 放 .ckpt 文件（只放一个，自动选最新的）
│   │   ├── sovits_weights\     # 放 .pth 文件（只放一个，自动选最新的）
│   │   ├── ref_audio\          # 参考音频（wav/flac，自动选第一个）
│   │   ├── ref_text.txt        # 参考文本（纯文本，一行）
│   │   └── alias.txt           # 【可选】角色标识，如写 "A"
│   └── 吐槽役\
│       ├── gpt_weights\
│       ├── sovits_weights\
│       ├── ref_audio\
│       ├── ref_text.txt
│       └── alias.txt           # 写 "B"
├── project_output\              # 工作产出
│   └── P111_四月新番导视\      # 项目文件夹（自动生成）

│       ├── 002.wav
│       ├── 003.wav
│       ├── sequence_log.txt    # 角色出现序号日志
│       ├── dialogue_order.txt  # 逐句顺序表
│       └── manifest.json       # 完整清单（JSON）
├── scripts\                     # 放文案文件（建议）
└── audio_synthesis_pipeline.py  # 自动化脚本
```

---

## 第一步：准备角色资产

每个角色一个文件夹，放在 `character_profile/` 下。

**必须包含：**
- `gpt_weights/xxx.ckpt` — GPT 模型权重
- `sovits_weights/xxx.pth` — SoVITS 模型权重
- `ref_audio/xxx.wav` — 参考音频（3~10秒）
- `ref_text.txt` — 参考音频对应的文本
- `ref_lang.txt` — 参考音频的语种（`中文` / `日文` / `英文` / `韩文` / `粤语`）

**可选：**
- `alias.txt` — 角色标识符。如果不放，默认用文件夹名。
  - 例如 `alias.txt` 里写 `A`，文案里就用 `[A]` 指代这个角色。

> 每个子文件夹只放一个权重文件即可，脚本自动选最新的。如果放了多个，会选修改时间最晚的那个。

---

## 第二步：准备文案

文案是纯文本 `.txt` 文件，编码 **UTF-8**，格式如下：

```text
[A-中文] 大家好，欢迎收看本期四月新番导视
[B-中文] 我是吐槽役，今天专门来泼冷水
[A-日文] こんにちは、今日は四月新番の紹介です
[B-中文] 这部作品啊，制作组经费明显不足
```

**规则：**
- 每行一句台词
- 以 `[角色标识-语言]` 开头，后面跟一个空格，然后是台词内容
- **角色标识** 必须和 `alias.txt`（或文件夹名）一致
- **语言** 可选，支持：`中文`、`日文`、`英文`、`韩文`、`粤语` 等
  - 如果省略语言（如 `[A]`），默认使用角色的 `ref_lang`
- 空行会被跳过

**多语言混合示例：**
```text
[A-日文] 私たちは自動化システムを使っています
[B-中文] 我们使用的是自动化系统
[A-日文] 長い文章でもちゃんと読めますか
[B-中文] 长文本也能正常朗读吗
```

---

## 第三步：运行脚本

打开 CMD 或 PowerShell，执行：

```cmd
cd /d D:\AIBLI\audio_synthesis
D:\AI音库\GPT-SoVITS-v2pro-20250604\runtime\python.exe audio_synthesis_pipeline.py --project P111 --script scripts\P111_script.txt
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `--project` | 是 | 项目编号/名称，如 `P111` |
| `--script` | 是 | 文案文件路径 |
| `--output_dir` | 否 | 输出根目录，默认 `project_output/` |
| `--top_k` | 否 | TTS 参数，默认 20 |
| `--top_p` | 否 | TTS 参数，默认 0.6 |
| `--temperature` | 否 | TTS 参数，默认 0.6 |
| `--speed` | 否 | 语速，默认 1.0 |
| `--pause` | 否 | 句内分段停顿（秒），默认 0.3 |

## 第四步：读取产出

项目跑完后，在 `project_output/P111/` 下会生成：

### 音频文件
`001.wav`, `002.wav`, `003.wav`... 按文案顺序命名。

### sequence_log.txt（角色次序日志）
```text
# 项目: P111
# 总句数: 6
# 角色出现次序
----------------------------------------
A: 1, 3, 5
B: 2, 4, 6
----------------------------------------
说话顺序: ABABAB
```

> 视频合成环节可以直接读这个文件，比如找 "A:" 后面的数字，就知道角色A在哪些序号出现了。

### dialogue_order.txt（逐句顺序表）
```text
001 [OK] [A] 大家好，欢迎收看本期四月新番导视
002 [OK] [B] 我是吐槽役，今天专门来泼冷水
...
```

### manifest.json（完整清单）
JSON 格式，包含每句的序号、角色、文本、生成状态。

## 已知限制

1. **必须分句生成**：GPT-SoVITS 本身对长文本需要切分，脚本已内置按标点自动切分+拼接。但每句台词如果超过一定长度，仍可能显存不足（单句建议控制在 50 字以内）。
2. **模型切换开销**：如果文案是 `ABBABB` 这种频繁交替，每次切换角色都会重新加载 GPT+SoVITS 权重，会有几秒停顿。建议同角色台词集中在一起更高效。
3. **参考音频长度**：必须 3~10 秒，否则 TTS 质量会下降或报错。
